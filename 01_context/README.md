# 01_context — Agent Context & Identity

This section defines **who the agent is**, what it knows, and how it presents itself. Context is the foundation of agent behavior — a well-defined context produces consistent, high-quality outputs.

## Contents

| File | Purpose |
|------|---------|
| `agent-identity.md` | Core identity definition: role, capabilities, personality, boundaries |
| `domain-glossary.md` | Domain-specific terminology and definitions |
| `persona-templates.md` | Reusable persona configurations for different use cases |

## Why Context Matters

The context window is the single most important lever for agent quality. Research consistently shows that:

- Agents with clear role definitions produce 40-60% fewer off-topic responses
- Domain glossaries reduce terminology confusion and improve precision
- Persona templates enable rapid deployment across different domains

## Usage

1. Start with `agent-identity.md` to establish the agent's core identity
2. Build `domain-glossary.md` as you encounter domain-specific terms
3. Create persona templates in `persona-templates.md` for repeatable configurations
